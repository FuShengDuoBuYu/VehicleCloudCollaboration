# 面向长尾场景触发的车–云协同智能驾驶研究

本仓库用于支撑毕业论文 / 科研研究，研究主题为：  
**面向智能驾驶的长尾（Long-Tail / Corner Case）场景判定与车–云协同触发机制**。

本项目**不以端到端自动驾驶为目标**，而是聚焦于一个更基础、也更关键的问题：

> 👉 **如何在车端准确判断“当前场景是否属于长尾场景”，并在必要时触发云端大模型进行进一步语义理解与推理。**

---

## 一、研究背景与动机

当前车载感知系统（如基于 YOLO 的轻量级模型）在**常规驾驶场景**中表现稳定，但在以下情况中容易失效或产生语义误判：

- 施工路段、异常交通组织
- 临时改道、非常规交通标志
- 感知结果之间存在**语义冲突**
- 数据分布外（Out-of-Distribution）场景

这些场景通常具有以下特征：

- **出现频率极低（Long-Tail）**
- **难以通过纯车端小模型可靠处理**
- **却对安全性高度敏感**

因此，本研究提出一种 **车–云协同的长尾场景触发框架**：
- 车端：负责高频、低成本的基础感知
- 云端：仅在必要时介入，提供复杂语义理解能力

---

## 二、问题定义

给定车载摄像头采集的单帧图像 \( I \)，设计一个 **长尾场景触发函数**：

```
f(I) -> {0, 1}
```

其中：
- `1`：判定为 **长尾 / Corner Case 场景**
- `0`：判定为 **常规驾驶场景**

⚠️ 本研究**不训练一个端到端分类模型**，而是通过多信号融合的方式构建 **可解释的触发机制**。

---

## 三、数据集设置

### 1️⃣ 长尾测试集（正样本）

- **CODA（Corner Case Dataset for Autonomous Driving）**
- 数据特点：
  - 人工精选的长尾 / 极端驾驶场景
  - 包含施工、异常交通组织等情况
- 使用方式：
  - **仅用于测试与评估**
  - 所有样本均视为长尾场景（Positive）

---

### 2️⃣ 非长尾测试集（负样本）

- **BDD100K（Normal Subset）**
- 构建方式：
  - 从 BDD100K 中筛选常规驾驶场景子集
  - 筛选条件包括：
    - 白天（daytime）
    - 晴天 / 多云（clear / cloudy）
    - 无施工、无事故

> 注：BDD100K 并不被假设为“完全不包含长尾场景”，  
> 而是作为 **常规驾驶分布（In-Distribution）的近似代表**。

### 🚗 车端（Edge）
- 硬件平台：**Raspberry Pi 5**
- 部署模型：
  - YOLOPv2：可行驶区域与车道线检测
  - YOLOv8n：行人、交通灯等目标检测
- 输出信息：
  - 感知结果
  - 置信度
  - 几何结构信息

### ☁️ 云端（Cloud）
- 大规模视觉 / 视觉语言模型（VLM）
- 提供：
  - 高层语义理解
  - 复杂场景推理
- 仅在**长尾触发条件满足时**被调用

## 五、长尾场景触发信号设计

<img width="1520" height="601" alt="image" src="https://github.com/user-attachments/assets/f1c60411-815a-462d-9f91-5a9ad766d1e1" />


长尾触发机制由多种信号共同决定：

### 1️⃣ 语义稀有性（Semantic Rarity）
- 从图像中提取语义关键词
- 与常规驾驶场景中的关键词频率分布进行对比
- 稀有关键词 → 更高长尾可能性

### 2️⃣ 模型不确定性（Model Uncertainty）
- 检测结果整体置信度偏低
- 目标数量或分布异常

### 3️⃣ 语义冲突（Semantic Conflict）
- 感知结果之间不一致
- 示例：
  - 隧道内识别到常规交通信号灯
  - 可行驶区域与交通控制语义冲突

最终通过融合上述信号，生成 **长尾触发分数** 并进行阈值判定。

---

## 六、评估方法与指标

本研究将长尾触发问题视为一个 **二分类判定系统**，评估指标包括：

### ✅ 长尾召回率（Recall on CODA）
- 衡量在 CODA 中成功触发的比例
- 体现系统对长尾场景的覆盖能力

### ⚠️ 误触发率（False Trigger Rate）
- 在 BDD100K-Normal 中被误判为长尾的比例
- 反映系统的稳定性与云端调用成本
